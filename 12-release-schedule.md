# Release Strategy

<v-clicks>

## Continuous Delivery

### Release Cadence

- **Minor Releases** - Every 4-6 weeks
- **Patch Releases** - As needed for critical fixes
- **Major Releases** - Annual, with breaking changes
- **Feature Previews** - Beta features for early adopters

### Version Numbering

- **Major.Minor.Patch** format (e.g., 39.3.0)
- Semantic versioning principles
- Clear changelog for each release
- Migration guides for major versions

</v-clicks>

# Old UI Phase-Out Plan

<v-clicks>

## Transition Timeline

### 2026 H1: Parallel Operation
- Both UIs available
- New UI as recommended option
- Feature gap closure continues
- User training and documentation

### 2026 H2: Transition Period
- New UI becomes default
- Old UI available but deprecated
- Migration assistance for users
- Final feature implementation

### 2027: Full Migration
- Old UI retired
- Full support for new UI only
- Complete documentation
- Community fully migrated
